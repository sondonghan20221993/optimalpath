
**결론**: N=16이 worst-case를 보장함 ✓

### 3. 형상 복잡도 vs 포화점 상관

**PCA Flatness와 포화점 N의 관계**:
- PCA 0.138 (box, 가장 평탄) → N=6
- PCA 0.167 (flat-simple) → N=6
- PCA 0.933 (sphere-small, 가장 구형) → N=8
- PCA 0.972 (sphere-large) → N=8

**피어슨 상관계수**: **r > 0.8** (강한 양의 상관)

**해석**: 형상이 복잡할수록 더 많은 등분 필요 → Orbit도 자동으로 형상에 응답

## 💡 해석

### 왜 Orbit이 더 강건한가?

1. **균등 분포의 강점**
   - 미지의 형상에 대해 "공평한" 시점 배치
   - 단일 구조 가정 없음 → 모든 형태에 적응

2. **Greedy의 약점**
   - Oracle이 점군 기반 (일부 영역의 점 밀도 낮으면 누락)
   - 평탄한 물체: 모든 시점이 동등하게 좋음 (Greedy 이득 없음)
   - 가려진 영역: Oracle 점군 자체가 관측 불가 (Greedy 무용)

3. **경합 조건의 공정성**
   - 같은 시점 예산 N → Greedy의 oracle 우위를 중화
   - 실제 환경(감지기 노이즈)에서는 oracle 성능 저하 가능

### 언제 Greedy가 나을까?

- **구형 물체 + 충분한 점군**: 특정 영역의 높은 정보량 활용 (10% gain)
- **예산 크면**: N > 12일 때 모두 100% → Greedy 이득도 포화
- **형상 사전지식 있을 때**: Oracle을 실제 감지기로 대체

### 발견의 의의

1. **알고리즘 무독립성**: Greedy oracle의 성능 우위가 상황에 따라 제한적
