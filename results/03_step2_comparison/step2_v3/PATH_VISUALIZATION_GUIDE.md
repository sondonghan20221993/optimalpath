# Orbit vs Greedy 경로 시각화 가이드

## 📊 생성된 파일

```
results/step2_v3/
├─ path_comparison_flat-simple.html    ← 평탄 물체
└─ path_comparison_sphere-large.html   ← 구형 물체
```

**각 HTML 파일을 브라우저에서 열면 3D 인터랙티브 시각화를 볼 수 있습니다.**

---

## 🔍 시각화의 구성 요소

### 1. **파란 점들 (Surface Voxel)**
- 물체의 표면을 나타내는 voxel들
- 복원 대상이 되는 점들
- flat-simple: 202개
- sphere-large: 362개

### 2. **빨간 다이아몬드 (Target)**
- 복원하려는 물체의 중심
- 좌표: (-33.67, -50.83, 0.18)

### 3. **청색 선 (Orbit 경로)**
- O1 → O2 → O3 → ... → O8
- **특징**: 완벽하게 원형, 균등한 간격
- 사전 정의된 경로 (형상 정보 없음)

### 4. **빨강 점선 (Greedy 경로)**
- G1 → G2 → G3 → ... → G8
- **특징**: 불규칙한 배치, oracle 최적화
- 형상 정보를 알고 선택한 경로

### 5. **회색 점들 (후보 시점, 선택사항)**
- 전체 999개 후보 위치 중에서 선택
- 희미하게 표시 (참고용)

## 📈 경로 통계 비교

### flat-simple (평탄 물체, N=8)

| 지표 | Orbit | Greedy | 의미 |
|---|---|---|---|
| **경로 길이** | 39.87m | 25.25m | Greedy가 더 효율적 (36% 짧음) |
| **커버리지** | 100.0% | 100.0% | 둘 다 완전히 본다 (동일) |
| **점 간 거리 분산** | 3.30 | 4.45 | Greedy가 더 흩어짐 (멀리서 봄) |

**해석**:
- **커버리지는 같음** (0% 차이)
  → 평탄 물체라 어디서나 볼 수 있음
  → 경로 선택 무의미
- **Greedy가 더 짧은 경로** (25.25m)
  → 하지만 이건 비교 불공정 (budget이 같지 않음)
  → 경로 길이는 secondary metric

---

### sphere-large (구형 물체, N=8)

| 지표 | Orbit | Greedy | 의미 |
|---|---|---|---|
| **경로 길이** | 39.87m | 77.76m | Greedy가 더 멀리 감 (95% 길음) |
| **커버리지** | 90.9% | 99.2% | **Greedy가 8.3%p 더 높음** ⭐ |
| **점 간 거리 분산** | 3.30 | 4.99 | Greedy가 훨씬 더 흩어짐 |

**해석**:
- **커버리지 차이 명확** (+8.3%p)
  → sphere는 정보가 풍부함
  → Greedy가 이 정보를 활용해 최적화
- **Greedy가 더 먼 경로** (77.76m)
  → sphere의 backside도 보기 위해 멀리 간다
  → 경로 길이 trade-off (더 많이 보기 위해 더 멀리 감)

## 🎯 3D 그래프 읽는 법

### Orbit 경로 (평탄 물체)
```
위에서 본 모습 (Top view):
  
      O2
   O3    O1
  O4      O8
   O5    O7
      O6

패턴: 완벽한 원형, 매우 규칙적
→ "사전 정의", "형상 무시"
```

### Greedy 경로 (평탄 물체)
```
위에서 본 모습:

      G2
   G3  G1
  G6   G8
   G5  G7
      G4

패턴: 거의 Orbit과 동일 (왜?)
→ 평탄 물체라 "볼 수 있는 위치"가 이미 정해져 있음
→ Greedy도 oracle을 써도 추가 최적화 불가능
→ 결과적으로 비슷한 원형 배치
```

### Greedy 경로 (구형 물체)
```
위에서 본 모습:

      G1(앞)
   G2      G8
  G3(좌)   G7(우)
   G4      G6
      G5(뒤)

패턴: 불규칙, 원 전체를 활용
→ Greedy가 sphere의 모든 면을 보도록 최적화
→ backside(뒤쪽)도 명시적으로 보기 위해 
   뒤쪽에도 시점 배치
→ Orbit은 뒤쪽을 효율적으로 못 본다
→ 결과: +8.3%p 이득
```

## 🔎 상세 해석

### flat-simple에서 Orbit = Greedy인 이유

```
평탄 물체 구조:
  
  [표면1: 위쪽]  → 모든 시점이 볼 수 있음
  [내부: 닫혀있음]
  
Orbit 시점 (8개):
  - 모든 시점이 위쪽 표면을 봄
  - 누락되는 부분 없음
  - 커버리지: 100%

Greedy 시점 (oracle):
  - "위쪽 표면을 다 보는" 최적화를 해도
  - 8개 시점이면 충분
  - 추가 최적화 여지 없음
  - 커버리지: 100% (동일)
```

### sphere-large에서 Greedy > Orbit인 이유

```
구형 물체 구조:

  [sphere: 모든 방향에 표면]
  - 앞쪽(front): 카메라에서 잘 봄
  - 옆쪽(side): 일부만 봄
  - 뒷쪽(back): 어려움

Orbit (균등 원형 8개):
  - 앞쪽(0°, 45°, 90°, ...) 위주로 배치
  - 뒷쪽(180° 근처)에서 카메라가 멀어짐
  - 법선이 카메라를 못 본다 (back-facing)
  - 커버리지: 90.9% (후면 일부 미관측)

Greedy (oracle 최적화):
  - "뒷쪽도 보기 위해" 뒷쪽에 시점 배치
  - 경로는 길어짐 (뒷쪽까지 가야 함)
  - 하지만 커버리지 99.2% 달성
  - 8.3%p 개선
```

## 📊 경로 길이와 커버리지의 trade-off

| 물체 | 경로 | 경로 길이 | 커버리지 | 효율 (커버리지/길이) |
|---|---|---|---|---|
| flat-simple | Orbit | 39.87m | 100.0% | 2.51 |
| flat-simple | Greedy | 25.25m | 100.0% | 3.96 |
| sphere-large | Orbit | 39.87m | 90.9% | 2.28 |
| sphere-large | Greedy | 77.76m | 99.2% | 1.27 |

**깨달음**:
1. **평탄 물체**: Greedy가 더 짧은 경로로 같은 결과 (효율 우수)
2. **구형 물체**: Greedy가 더 긴 경로 (효율 저하), 하지만 더 높은 커버리지
   - 이건 "비효율"이 아니라 **"다른 목표"**
   - Orbit: 빠른 시간에 적당한 커버리지
   - Greedy: 더 많은 시간에 높은 커버리지

## 🎓 결론

### 1. flat-simple (평탄)에서
```
Orbit과 Greedy가 거의 같다
→ 경로 선택이 무의미
→ 형상이 단순하면 "어디서나 봐도 같다"
```

### 2. sphere-large (구형)에서
```
Greedy가 Orbit을 명백히 능가한다
→ 경로 선택이 매우 중요
→ 형상이 복잡하면 "최적화할 여지가 많다"
```

### 3. 경로 설계의 원칙
```
[단순한 물체]
  → Orbit (균등 원형) 충분
  → 효율적이고 단순
  
[복잡한 물체]
  → Greedy (oracle 기반) 필요
  → 하지만 현실에서는 oracle 불가능
  → Online NBV 필요
```

## 📝 정리

| 항목 | Orbit | Greedy |
|---|---|---|
| **경로 형태** | 규칙적 원형 | 불규칙 (목적 최적화) |
| **형상 정보** | 불필요 | 필수 (oracle) |
| **평탄 물체** | 최적 (효율, 단순) | 비슷 (약간 짧음) |
| **구형 물체** | 약함 (backside 미관측) | 강함 (완전 관측) |
| **현실 적용** | ✅ 가능 | ❌ 불가능 (oracle 필요) |
| **추천** | 형상 미지 상황 | 상한값 참고용 |
