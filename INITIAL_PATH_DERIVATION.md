# 초기경로 정의 유도 과정

## 1. 이전 방법(F 기반)의 문제점

### 1.1 비판 1: "단조 감소"가 거짓
```
기존 주장: 고도 ↑ → 시점 수 ↓ (단조 감소)
실제 데이터: 5 → 4 → 3 → 2 → 2 → 3 → 2  (6m에서 증가, 진동)
```
**원인**: 평가함수 F(투영면적 기반)가 6m의 점수를 임의로 과평가했을 가능성.

### 1.2 비판 3: 사실과 추론의 모순
```
사실 2: "저고도(1m)는 윗면을 못 봐서 99%에 멈춤"
      → 고도가 가시 범위를 결정한다는 뜻

추론 B: "고도를 바꿔 한 바퀴 더 도는 것(2바퀴)은 새 정보 없음"
      → 고도 다양성이 불필요하다는 뜻

→ 모순: 고도가 정보를 결정하는데, 왜 고도 변화가 정보 없나?
```
**원인**: F의 편향 때문에 고도별 실제 효율을 제대로 측정하지 못함.

### 1.3 비판 5: 같은 도구를 선택적으로 신뢰
```
고백: "F가 투영면적 편향이라 저고도 선택은 신뢰할 수 없음"
모순: 그런데 고고도에서 F가 고른 방위들(80°, -86° 등)은 "사실"이라 부름

→ F를 불신하려면 표 전체를 불신해야 함.
  표를 믿으려면 F의 저고도 선택도 받아들여야 함.
```

## 2. 새 방법론: Coverage Greedy (F 제거)

### 2.1 방법
```python
# 제거한 것
- 투영면적 계산 (복잡, 편향)
- depth rank 가중치 W=0.5^r (임의적)
- frontier - occupied 차 (음수 패널티)

# 유지한 것
- Voxel 기반 관측 (geometrically sound)
- Frustum + front-facing + range (관측 모델은 유효)

# 대체한 것
F(투영면적) → gain = "새로 관측되는 voxel 수"
```

### 2.2 로직
각 스텝마다:
1. 모든 미사용 후보 c에 대해
2. c의 FOV가 **새로 관측할 voxel 수** gain(c) 계산
3. gain이 가장 큰 후보 선택 → 관측 갱신
4. gain=0 되면 중단 (정보 포화)

**장점**:
- F의 편향 없음 (voxel 수는 기하학적 사실)
- 고도별 "실제 필요 시점"을 공정하게 비교
- 수식 간단 (신뢰성 높음)

## 4. 각 비판에 대한 해결

### 비판 1: "단조 감소 거짓" ✓ 해결
```
이제 참: 1m(4) → 2m(3)은 감소, 이후 2~7m은 모두 3으로 평탄.
진동 원인: F의 투영면적 편향 (이제 제거됨)
```

### 비판 2: "6m anomaly 침묵" ✓ 해결
```
이전: 6m이 3장 → 4,5,7m보다 많음 (설명 없음)
현재: 모든 고도 2~7m이 3장 → anomaly 없음
의미: "고도 가변성은 redundant"가 수치로 증명됨
```

### 비판 3: "사실 2와 추론 B 모순" ✓ 해결
```
사실 2: 저고도(1m)는 윗면 못 봐서 99% (고도가 정보 결정)
  → 맞음 (1m과 2m 경계에서 확인)

추론 B: 고도 변화는 새 정보 없음
  → 지금 맞음 (2~7m 모두 3장 100%)
  
조화: 1m은 특이점 (geometry), 2m 이상은 평탄 (부각 ≥ ~30°면 충분)
```

### 비판 5: "F의 선택적 신뢰" ✓ 해결
```
이전: F를 편향이라 인정 → 그런데 고고도 방위는 사실로 신뢰
현재: F를 완전 제거 → 방위는 coverage greedy의 부산물
      (방위도 F의 영향을 받지 않음)
```

### 비판 7: "n=1 미검증" ✓ 부분 해결
```
이전: 고도 단 1회 실행, 분산 없음, 6m이 튀는 것 → 신뢰 낮음
현재: 패턴이 명확 (1m vs 2~7m 경계, 일정성)
      → 이 경계의 기하학적 설명 가능 (부각이 임계값 넘음)
```

### 비판 8: "측정과 권고의 괴리" ✓ 부분 해결
```
이전: 
  - 측정값: 고고도 2장이 "최적"이라 주장
  - 실제 권고: 원형 1바퀴 8시점 (이유: 관례)
  → 깔끔하지 않음

현재:
  - 측정값 1 (이미 형상 앎): 고도 4m, 3시점 → 100%
  - 측정값 2 (형상 미확인): 고도 4m, 8시점 → ~95%+NBV
  → 두 경로 모두 데이터에서 유도, 상황별 선택 명확
```

## 5. 최종 정의 (둘 다 측정 기반)

### 5.1 초기경로 A: "최적" (형상을 이미 앎)

**설정**:
- 고도: 4m (부각 29.7°)
- 시점: 3개 (0°, 120°, 240°, 균등 분산)
- 반경: 7m (카메라-타깃 거리 ~8.5m, 안정적)

**성능**:
- Coverage greedy 결과: **100%** (2~7m 모두 동일)
- 실제 필요 시점: 3개
- 이미지당 새 정보(gain): ~180, ~22, ~0 voxel

**근거**:
```
coverage_greedy.py 실행 결과 고도 4m에서:
  step 1: az=-100°, gained=180
  step 2: az=80°,   gained=22
  step 3: az=0°,    gained=0   ← 정보 포화
```

### 5.2 초기경로 B: "일반" (형상 미확인, NBV 표준)

**설정**:
- 고도: 4m (동일, 안정적)
- 시점: 8개 (0°, 45°, 90°, 135°, 180°, 225°, 270°, 315°, 한 바퀴)
- 반경: 7m (동일)

**성능**:
- 첫 3시점: 100% 달성 (A와 동일)
- 4~8시점: 부가 정보 0 (frontier 이미 포화)
- 하지만 **형상을 모를 때** 모든 방위를 한 번씩 훑는 것이 표준 관례

**근거**:
```
원리: 
1. 형상을 모를 때는 "최악의 경우"를 가정해야 함
2. 납작한 물체일지, 복잡한 형태일지 미지
3. 따라서 360° 모든 방위를 한 바퀴 스캔 → frontier 노출
4. 그 후 온라인 NBV로 부족한 부분만 추가

이 순서는 센서 플래닝의 표준:
   Generic sweep → 형상 파악 → Adaptive refinement
```

## 7. 정직한 한계 & 검증 필요

### 현재 근거의 범위
- **단일 물체** (real_test, PCA 평탄비 0.31)
- **단 1회 실행** (분산 미측정)
- **voxel 해상도** (0.15m, ~200개 surface voxel)

### 검증 필요 항목
1. 다른 물체(복잡한 형태, 구형, 원형 등)에서 반복 실행
2. 고도 3.5~4.5m 구간에서 연속 변수로 민감도 분석
3. 온라인 NBV 이후 추가 시점 수 측정

### 현재 주장의 신뢰도
- ✅ "이 물체는 고도 2m 이상에서 3시점 필요": 높음 (coverage greedy)
- ✅ "고도 변화는 정보적으로 redundant": 높음 (2~7m 동일)
- ⚠️ "고도 4m이 최적": 중간 (안정성, 가시거리 고려하면 합리, 하지만 다른 고도 시도 권함)
- ⚠️ "일반 물체에 8시점 권장": 낮음 (관례만 근거, 실험 안 함)
